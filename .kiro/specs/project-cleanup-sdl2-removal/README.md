# SDL2 Cleanup System

Автоматизована система для безпечного очищення проекту "Punic Wars: Castra" від застарілих файлів та документації, пов'язаних з невдалою міграцією на SDL2.

## Огляд

Проект повністю працює на Raylib, але містить залишки документації про спробу міграції на SDL2, яка не була завершена. Ця система автоматично видаляє всі непотрібні файли та згадки про SDL2, зберігаючи при цьому повну працездатність поточної версії гри.

## Можливості

- ✅ Автоматичне видалення SDL2-специфічних файлів
- ✅ Очищення документації від SDL2 згадок
- ✅ Видалення SDL2 гілки з локального та віддаленого репозиторію
- ✅ Створення резервних копій перед модифікацією
- ✅ Валідація проекту після очищення
- ✅ Детальний звіт про виконані операції
- ✅ Можливість відкату змін у разі проблем

## Швидкий старт

### 1. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 2. Перегляд що буде зроблено (Dry Run)

```bash
python cleanup.py --dry-run
```

### 3. Виконання очищення

```bash
python cleanup.py
```

### 4. Перегляд звіту

Після виконання звіт буде збережено в `.kiro/specs/project-cleanup-sdl2-removal/cleanup_report.md`

## Використання

### Базове використання

```bash
# Звичайне виконання з підтвердженням
python cleanup.py

# Показати що буде зроблено без виконання
python cleanup.py --dry-run

# Детальний вивід
python cleanup.py --verbose

# Пропустити видалення git гілки
python cleanup.py --skip-git
```

### Опції командного рядка

- `--dry-run` - Показати що буде зроблено без виконання змін
- `--no-backup` - Пропустити створення резервних копій (небезпечно!)
- `--skip-git` - Пропустити видалення SDL2 гілки
- `--verbose` - Детальний вивід
- `-h, --help` - Показати довідку

### Приклади

```bash
# Перевірити що буде зроблено
python cleanup.py --dry-run

# Виконати очищення з детальним виводом
python cleanup.py --verbose

# Очистити тільки файли, без git операцій
python cleanup.py --skip-git

# Виконати без резервних копій (не рекомендується!)
python cleanup.py --no-backup
```

## Що буде видалено

### Файли

- `cpp/SDL2_MIGRATION_TODO.md` - TODO список міграції на SDL2
- `cpp/QUICK_START_SDL2.md` - Швидкий старт для SDL2

### Документація

Згадки про SDL2 будуть видалені з:
- `ТЗ від грока`
- `Переписа з грок вихідні дані.txt`
- `cpp/README.md`
- `SESSION_CONTEXT.md`

### Git гілки

- Локальна гілка `SDL2`
- Віддалена гілка `origin/SDL2`

## Безпека

### Резервні копії

Система автоматично створює:
1. **Safety commit** - git commit з поточним станом проекту
2. **Резервні копії файлів** - копії всіх файлів, що будуть модифіковані

### Відкат змін

Якщо щось піде не так, система запропонує відкотити зміни:

```python
# Автоматично при помилці
orchestrator.rollback_changes()
```

### Валідація

Після очищення система перевіряє:
- ✓ Наявність всіх критичних файлів
- ✓ Відсутність SDL2 включень у коді
- ✓ Цілісність документації
- ✓ Працездатність проекту

## Архітектура

Система складається з 6 компонентів:

```
┌─────────────────────────────────────────────────────────────┐
│                    Cleanup Orchestrator                      │
│                  (Координує всі операції)                    │
└──────────────────────┬──────────────────────────────────────┘
                       │
         ┌─────────────┼─────────────┬─────────────┐
         │             │             │             │
         ▼             ▼             ▼             ▼
┌────────────┐  ┌────────────┐  ┌────────────┐  ┌────────────┐
│   Backup   │  │    File    │  │    Git     │  │ Validation │
│  Manager   │  │  Cleaner   │  │  Manager   │  │  Engine    │
└────────────┘  └────────────┘  └────────────┘  └────────────┘
         │             │             │             │
         └─────────────┴─────────────┴─────────────┘
                       │
                       ▼
              ┌────────────────┐
              │     Report     │
              │   Generator    │
              └────────────────┘
```

### Компоненти

1. **Backup Manager** - Створює резервні копії та відкочує зміни
2. **File Cleaner** - Видаляє файли та очищає документацію
3. **Git Manager** - Керує git операціями
4. **Validation Engine** - Перевіряє працездатність проекту
5. **Report Generator** - Створює детальний звіт
6. **Cleanup Orchestrator** - Координує всі операції

## Часті питання

**Q: Чи безпечно запускати цей скрипт?**
A: Так, система створює резервні копії та safety commit перед будь-якими змінами.

**Q: Що робити якщо щось пішло не так?**
A: Система автоматично запропонує відкотити зміни. Також можна вручну відкотитися до safety commit.

**Q: Чи можна запустити тільки частину операцій?**
A: Так, використовуйте `--skip-git` щоб пропустити git операції.

**Q: Де знаходяться резервні копії?**
A: У `.kiro/specs/project-cleanup-sdl2-removal/backups/`

**Q: Як переглянути що буде зроблено?**
A: Використайте `--dry-run` режим.
