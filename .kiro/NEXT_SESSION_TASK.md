# Задача для наступної сесії: Інтеграція генератора мапи

## 🎯 Головна мета

Синхронізувати геометрію ізометричної мапи з областями взаємодії (pathfinding, mouse clicks, building placement).

## ❗ Проблема

Зараз генератор мапи працює окремо, і його дані не використовуються для визначення областей прохідності та взаємодії. Чорні лінії тайлів не відповідають візуальній геометрії мапи.

## 📋 План дій

### 1. Аналіз поточного стану
- [ ] Перевірити, як TileMap використовується в грі
- [ ] Перевірити, як NavigationGrid обчислює прохідність
- [ ] Знайти розбіжності між системами координат

### 2. Синхронізація систем координат
- [ ] Переконатися, що CoordinateConverter (128x64) використовується скрізь
- [ ] Оновити NavigationGrid для використання ізометричної геометрії
- [ ] Синхронізувати mouse interaction з TileMap

### 3. Інтеграція TileMap з pathfinding
- [ ] Використовувати TileMap.isPassable() для перевірки прохідності
- [ ] Оновити NavigationGrid для використання даних TileMap
- [ ] Тестувати рух юнітів по ізометричній сітці

### 4. Виправлення texture_offset будівель
- [ ] Експериментувати з різними значеннями
- [ ] Або змінити підхід (anchor point замість offset)
- [ ] Тестувати візуально

## 🔧 Ключові файли

- `cpp/src/tilemap/coordinates.h` - CoordinateConverter (128x64)
- `cpp/src/tilemap/tilemap.h` - TileMap структура
- `cpp/src/pathfinding.h` - NavigationGrid (16x16 - ПРОБЛЕМА!)
- `cpp/src/main.cpp` - mouse interaction, building placement
- `cpp/src/building.h` - texture_offset (-160)

## 📊 Поточні параметри

- **Тайл**: 128x64 пікселів
- **NavigationGrid**: 16x16 пікселів (НЕ синхронізовано!)
- **Мапа**: 80x80 тайлів
- **texture_offset**: {-192, -160}

## ✅ Що вже працює

- Тайлсет завантажується (256x128)
- Спрайти будівель завантажуються (Praetorium, Skene)
- Спрайти юнітів готові (4 файли)
- Pathfinding використовує GridCoords
- Координати ресурсів виправлені (280, 580)

## ❌ Що не працює

- Будівлі не видно на мапі (texture_offset)
- Геометрія мапи не відповідає областям взаємодії
- NavigationGrid використовує прямокутну сітку 16x16 замість ізометричної

## 💡 Підказки

1. Можливо потрібно створити новий spec для цієї задачі
2. Це може бути bugfix (виправлення геометрії) або feature (інтеграція генератора)
3. Почати з аналізу - як зараз працює NavigationGrid
4. Перевірити, чи TileMap.isPassable() викликається
