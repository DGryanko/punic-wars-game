## Зміни

### 1. Виправлення логіки збору ресурсів

#### Проблеми, які були виправлені:

**Раби не йшли здавати ресурси після збору:**
- Додано автоматичну зупинку збору (`stopHarvesting()`) коли раб повний
- Додано автоматичний рух до будівлі для здачі ресурсів
- Виправлено логіку перевірки відстані до будівлі

**Пріоритет здачі ресурсів:**
- Раби тепер здають ресурси спочатку в Квесторіум (QUESTORIUM_ROME)
- Якщо Квесторіум не знайдено, використовується HQ
- Це відповідає історичній точності (квесторій = скарбниця)

**Зменшена швидкість збору:**
- Швидкість збору зменшена з 2 до 1 одиниці за кадр (в 2 рази)
- Це робить гру більш збалансованою

#### Код змін:

```cpp
// В unit.h - автоматична зупинка збору при повному інвентарі
void addResources(int food, int gold) {
    // ... додавання ресурсів ...
    
    // Якщо повний, зупинити збір

        stopHarvesting();
    }
}

// В main.cpp - зупинка збору перед рухом до будівлі

    unit.stopHarvesting(); // ВАЖЛИВО: зупинити збір перед рухом
    // ... логіка руху до будівлі ...
```

### 2. Базовий Pathfinding (обхід перешкод)

#### Проблема:
Юніти застрягали між будівлями і не могли знайти шлях навколо перешкод.

#### Рішення:
Додано простий алгоритм обходу перешкод:

1. **При колізії юніт не зупиняється**, а пробує знайти альтернативний шлях
2. **Тестуються 4 напрямки**: +45°, -45°, +90°, -90° від поточного напрямку
3. **Вибирається перший вільний напрямок** без колізій
4. **Якщо всі напрямки заблоковані**, тоді юніт зупиняється

#### Алгоритм:

```cpp
if (hasCollision) {
    units[i].x = old_x;
    units[i].y = old_y;
    
    if (units[i].is_moving) {
        // Обчислюємо кут до цілі
        float angle = atan2(dy, dx);
        
        // Пробуємо 4 напрямки
        float angles[] = {
            angle + 0.785f,  // +45°
            angle - 0.785f,  // -45°
            angle + 1.57f,   // +90°
            angle - 1.57f    // -90°
        };
        
        // Тестуємо кожен напрямок
        for (float testAngle : angles) {
            // Обчислюємо тестову позицію
            int test_x = x + cos(testAngle) * speed * 2;
            int test_y = y + sin(testAngle) * speed * 2;
            
            // Перевіряємо колізії
            if (!hasCollision(test_x, test_y)) {
                // Рухаємося в цьому напрямку
                x = test_x;
                y = test_y;
                break;
            }
        }
    }
}
```

### 3. Покращення діагностики

Додано printf для відстеження:
- Призначення рабів до ресурсів
- Здачі ресурсів
- Руху до будівель

## Як це працює

### Збір ресурсів (повний цикл):

1. **Призначення**: Гравець клікає ПКМ по ресурсу
   - Раб отримує координати ресурсу
   - Раб отримує координати найближчого Квесторіуму
   - Раб починає рух до ресурсу

2. **Збір**: Раб досягає ресурсу
   - Починає збір (1 одиниця за кадр)
   - Індикатор заповнення показує прогрес
   - При досягненні максимуму (15 одиниць) збір автоматично зупиняється

3. **Здача**: Раб автоматично йде до Квесторіуму
   - Використовує pathfinding для обходу перешкод
   - При досягненні будівлі здає ресурси
   - Ресурси додаються до фракції

4. **Повернення**: Раб автоматично повертається до ресурсу
   - Цикл повторюється до вичерпання ресурсу

### Pathfinding:

1. **Юніт рухається до цілі**
2. **При зіткненні з перешкодою**:
   - Обчислюється кут до цілі
   - Тестуються 4 альтернативні напрямки
   - Вибирається перший вільний напрямок
3. **Юніт продовжує рух** в новому напрямку
4. **Поступово обходить перешкоду** і досягає цілі

## Відомі обмеження

### Pathfinding:
- Це **базовий алгоритм**, не повноцінний A*
- Може не знайти оптимальний шлях у складних лабіринтах
- Працює добре для простих перешкод (будівлі)

- [ ] Додати формації

## Використання

### Тестування збору ресурсів:
1. Створіть рабів (SLV кнопка в HQ)
2. Виділіть рабів
3. ПКМ по ресурсу (зелений або жовтий квадрат)
4. Спостерігайте за автоматичним циклом

### Тестування pathfinding:
1. Створіть юнітів
2. Відправте їх між будівлями
3. Спостерігайте як вони обходять перешкоди
4. Юніти не повинні застрягати

## Технічні деталі

### Продуктивність:
- Pathfinding додає мінімальне навантаження
- Тестується максимум 4 напрямки при колізії
- Немає складних обчислень

### Математика:
- Використовується `atan2()` для обчислення кута
- Використовується `cos()` та `sin()` для обчислення напрямку
- Кути в радіанах: 45° = 0.785, 90° = 1.57

### Константи:
```cpp
const float ANGLE_45 = 0.785f;   // 45 градусів
const float ANGLE_90 = 1.57f;    // 90 градусів
```
