# Виправлення помилок - Система будівництва рабом

## Виправлені проблеми

### 1. Будівлі будувалися одна на одній ❌ → ✅
**Проблема:** Будівлі могли будуватися впритул одна до одної без відступу.

**Рішення:** Змінено логіку `findFreeBuildSpot()` в `slave_build_panel.h`:
- Тепер перевіряється область 3x3 тайли (2x2 будівля + 1 тайл відступу з кожного боку)
- Це гарантує мінімум 1 тайл відступу між будівлями

```cpp
// Було: перевірка тільки 2x2 тайли
for (int r = 0; r < 2 && isFree; r++) {
    for (int c = 0; c < 2 && isFree; c++) {
        // ...
    }
}

// Стало: перевірка 3x3 тайли (з відступом)
for (int r = -1; r < 3 && isFree; r++) {
    for (int c = -1; c < 3 && isFree; c++) {
        // ...
    }
}
```

### 2. Юніти проходили крізь будівлі ❌ → ✅
**Проблема:** Після будівництва нової будівлі pathfinding grid не оновлювався, тому юніти не знали про нову перешкоду.

**Рішення:** Додано оновлення pathfinding grid після кожної нової будівлі:

**Файл:** `slave_build_panel.h`
```cpp
// Додано include
#include "pathfinding.h"

// Додано зовнішню змінну
extern PathfindingManager pathfindingManager;

// В методі spawnBuilding():
buildings->push_back(newBuilding);

// ВАЖЛИВО: Оновлюємо pathfinding grid після додавання будівлі
pathfindingManager.updateGrid(*buildings);
printf("[SlaveBuildPanel] Pathfinding grid updated after building placement\n");
```

Тепер юніти будуть обходити нові будівлі!

### 3. Раб не відносив ресурси в квесторій ❌ → ✅
**Проблема:** Раб набирав ресурси, але не йшов здавати їх в квесторій. Проблема була в тому, що:
1. Використовувалися застарілі screen координати замість grid
2. Не було динамічного пошуку найближчої будівлі для здачі

**Рішення:** Повністю переписано логіку здачі ресурсів в `ProcessResourceHarvesting()`:

**Файл:** `cpp/src/main.cpp`

**Було:**
```cpp
// Використовувалися збережені screen координати
float distToBuilding = sqrt(pow(unit.x - unit.dropoff_building_x, 2) + 
                           pow(unit.y - unit.dropoff_building_y, 2));
```

**Стало:**
```cpp
// Динамічний пошук найближчої будівлі (квесторіум або HQ)
Building* dropoffBuilding = nullptr;
float minDist = 999999.0f;

// Спочатку шукаємо квесторіум
for (auto& building : buildings) {
    if (building.faction == playerFaction && building.type == QUESTORIUM_ROME) {
        GridCoords unitPos = unit.getGridPosition();
        GridCoords buildingPos = building.getGridPosition();
        float dist = sqrt(pow(unitPos.row - buildingPos.row, 2) + 
                        pow(unitPos.col - buildingPos.col, 2));
        if (dist < minDist) {
            minDist = dist;
            dropoffBuilding = &building;
        }
    }
}

// Якщо не знайшли квесторіум, шукаємо HQ
if (!dropoffBuilding) {
    // ... аналогічний пошук HQ
}
```

**Переваги нового підходу:**
- ✅ Завжди знаходить найближчу будівлю для здачі
- ✅ Пріоритет квесторіуму над HQ
- ✅ Використовує grid координати (точніше)
- ✅ Працює з будь-якою кількістю квесторіумів
- ✅ Детальне логування для дебагу

## Тестування

### Як перевірити виправлення:

**Проблема 1 - Відступ між будівлями:**
1. Створіть раба
2. Побудуйте квесторіум
3. Побудуйте контуберній
4. Перевірте що між ними є мінімум 1 тайл відступу

**Проблема 2 - Обхід будівель:**
1. Побудуйте кілька будівель
2. Виберіть юніта
3. Накажіть йому рухатися через будівлі (ПКМ)
4. Перевірте що юніт обходить будівлі, а не проходить крізь них

**Проблема 3 - Здача ресурсів:**
1. Побудуйте квесторіум
2. Виберіть раба
3. Накажіть йому збирати ресурси (ПКМ по жовтій/синій точці)
4. Зачекайте поки раб наповниться
5. Перевірте що раб автоматично йде до квесторіуму
6. Перевірте що ресурси додаються до загальних запасів
7. Перевірте що раб повертається до ресурсу після здачі

## Технічні деталі

### Змінені файли:
1. `cpp/src/slave_build_panel.h`:
   - Метод `findFreeBuildSpot()` - додано перевірку 3x3 області
   - Метод `spawnBuilding()` - додано оновлення pathfinding
   - Додано include `pathfinding.h`
   - Додано extern `pathfindingManager`

2. `cpp/src/main.cpp`:
   - Функція `ProcessResourceHarvesting()` - повністю переписано логіку здачі ресурсів
   - Використання grid координат замість screen
   - Динамічний пошук найближчої будівлі

### Константи:
- Відступ між будівлями: **1 тайл** (перевіряється 3x3 область)
- Відстань здачі ресурсів: **3 тайли** (було 50 пікселів)
- Частота дебаг логів: **60 кадрів** (1 секунда при 60 FPS)

## Відомі обмеження

- Раби все ще не мають анімації ходьби до будівлі
- Немає візуального індикатора куди раб йде здавати ресурси
- Якщо квесторіум знищено під час руху раба, він може застрягти
