# 🎨 Тайлсет для ізометричної карти

## 📍 Розташування файлу

**Файл повинен бути тут**: `cpp/assets/isometric_tileset.png`

## 🚀 Швидкий старт

### Варіант 1: Згенерувати простий placeholder (рекомендовано для тестування)

```bash
cd cpp
python generate_tileset.py
```

Це створить простий тайлсет з суцільними кольорами для тестування.

### Варіант 2: Використати свій тайлсет

1. Створіть файл `isometric_tileset.png` розміром 256x128 пікселів
2. Розташуйте 4 ізометричні тайли (128x64 кожен) у сітці 2x2
3. Збережіть в `cpp/assets/isometric_tileset.png`

Детальні інструкції: `TILESET_INSTRUCTIONS.md`

## 📐 Технічні вимоги

- **Розмір файлу**: 256x128 пікселів
- **Формат**: PNG з прозорістю
- **Структура**: 4 тайли по 128x64 у сітці 2x2

```
┌─────────────┬─────────────┐
│   Трава     │    Вода     │
│  (0, 0)     │  (128, 0)   │
├─────────────┼─────────────┤
│   Пісок     │   Дорога    │
│  (0, 64)    │  (128, 64)  │
└─────────────┴─────────────┘
```

## 🎨 Що малювати

### 1. Трава (0, 0) - Зелений
- Прохідна місцевість
- Нормальна швидкість
- Приклад: Зелене поле

### 2. Вода (128, 0) - Синій
- Непрохідна місцевість
- Приклад: Річка, озеро

### 3. Пісок (0, 64) - Бежевий
- Прохідна місцевість
- Нормальна швидкість
- Приклад: Пляж, пустеля

### 4. Дорога (128, 64) - Коричневий
- Прохідна місцевість
- Збільшена швидкість (1.5x)
- Приклад: Римська дорога

## 🔍 Форма тайлу

Кожен тайл має форму **ромба (діаманта)**:

```
        (64, 0)
          ╱╲
         ╱  ╲
        ╱    ╲
(0,32) ╱      ╲ (128,32)
       ╲      ╱
        ╲    ╱
         ╲  ╱
          ╲╱
        (64, 64)
```

## ✅ Перевірка

Після створення тайлсету запустіть тест:

```bash
cd cpp
tilemap_test.exe
```

Якщо файл не знайдено, генератор працюватиме в **debug режимі** (кольорові ромби).
