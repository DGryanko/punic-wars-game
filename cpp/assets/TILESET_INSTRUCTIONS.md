# 🎨 Інструкція: Як створити тайлсет для генератора

## 📐 Технічні вимоги

### Розмір файлу
- **Назва файлу**: `isometric_tileset.png`
- **Розташування**: `cpp/assets/isometric_tileset.png`
- **Розмір**: 256x128 пікселів
- **Формат**: PNG з прозорістю (альфа-канал)

### Структура тайлсету

Тайлсет складається з **4 ізометричних тайлів** розміром 128x64 пікселів кожен, розташованих у сітці 2x2:

```
┌─────────────┬─────────────┐
│   Трава     │    Вода     │  ← Верхній ряд (y=0)
│  128x64     │   128x64    │
│  (0, 0)     │  (128, 0)   │
├─────────────┼─────────────┤
│   Пісок     │   Дорога    │  ← Нижній ряд (y=64)
│  128x64     │   128x64    │
│  (0, 64)    │  (128, 64)  │
└─────────────┴─────────────┘
     ↑              ↑
   x=0           x=128
```

## 🎨 Форма ізометричного тайлу

Кожен тайл має форму **ромба (діаманта)** з такими параметрами:

```
        Верхня точка (64, 0)
              ╱╲
             ╱  ╲
            ╱    ╲
           ╱      ╲
Ліва (0,32)      Права (128,32)
           ╲      ╱
            ╲    ╱
             ╲  ╱
              ╲╱
       Нижня точка (64, 64)
```

### Координати вершин ромба:
- **Верх**: (64, 0)
- **Право**: (128, 32)
- **Низ**: (64, 64)
- **Ліво**: (0, 32)

### Співвідношення сторін:
- **Ширина**: 128 пікселів
- **Висота**: 64 пікселі
- **Ratio**: 2:1 (ізометрична проекція)

## 🖼️ Опис кожного тайлу

### 1. Трава (Grass) - Позиція (0, 0)
- **Колір**: Зелений (#4CAF50 або подібний)
- **Текстура**: Трава, рослинність
- **Властивості**: Прохідна, нормальна швидкість
- **Приклад**: Зелене поле з травою

### 2. Вода (Water) - Позиція (128, 0)
- **Колір**: Синій (#2196F3 або подібний)
- **Текстура**: Вода, хвилі
- **Властивості**: Непрохідна
- **Приклад**: Річка, озеро, море

### 3. Пісок (Sand) - Позиція (0, 64)
- **Колір**: Бежевий (#F5DEB3 або подібний)
- **Текстура**: Пісок, пустеля
- **Властивості**: Прохідна, нормальна швидкість
- **Приклад**: Пляж, пустеля

### 4. Дорога (Road) - Позиція (128, 64)
- **Колір**: Коричневий (#8B4513 або подібний)
- **Текстура**: Бруківка, дорога
- **Властивості**: Прохідна, збільшена швидкість (1.5x)
- **Приклад**: Римська дорога, шлях

## 🛠️ Як створити тайлсет

### Варіант 1: Photoshop / GIMP

1. Створіть новий файл 256x128 пікселів
2. Розділіть на 4 квадрати 128x64
3. В кожному квадраті намалюйте ромб:
   - Використайте інструмент "Polygon" (4 сторони)
   - Або намалюйте вручну за координатами вершин
4. Заповніть кожен ромб відповідною текстурою
5. Збережіть як PNG

### Варіант 2: Aseprite (для піксель-арту)

1. Створіть файл 256x128
2. Намалюйте 4 ізометричні тайли
3. Використайте піксель-арт стиль
4. Експортуйте як PNG

### Варіант 3: Використати готові тайли

Якщо у вас є готові ізометричні тайли:
1. Відкрийте їх в редакторі
2. Масштабуйте до 128x64 кожен
3. Розташуйте в сітці 2x2
4. Збережіть як один файл 256x128

## 🎨 Рекомендації по стилю

### Колірна палітра:
- **Трава**: Зелені відтінки (#4CAF50, #8BC34A, #CDDC39)
- **Вода**: Сині відтінки (#2196F3, #03A9F4, #00BCD4)
- **Пісок**: Бежеві відтінки (#F5DEB3, #FFE4B5, #FFDEAD)
- **Дорога**: Коричневі відтінки (#8B4513, #A0522D, #D2691E)

### Деталізація:
- Додайте текстури (трава, хвилі, камінці)
- Використайте градієнти для об'єму

### Стиль:
- **Піксель-арт**: Чіткі пікселі, обмежена палітра
- **Hand-drawn**: Малюнок від руки
- **Realistic**: Реалістичні текстури
- **Cartoon**: Мультяшний стиль

## 🚀 Тестування тайлсету

Після створення тайлсету:

1. Збережіть файл як `cpp/assets/isometric_tileset.png`
2. Запустіть тестовий додаток:
   ```bash
   cd cpp
   tilemap_test.exe
   ```
3. Перевірте, що всі тайли відображаються правильно
4. Натисніть SPACE для генерації нової карти

## 🔧 Якщо тайлсет не працює

### Debug режим:
Якщо файл `isometric_tileset.png` не знайдено, генератор автоматично перейде в **debug режим** та малюватиме кольорові ромби:
- Зелений = Трава
- Синій = Вода
- Бежевий = Пісок
- Коричневий = Дорога

Це дозволяє тестувати генератор навіть без тайлсету!

## 📦 Приклад простого тайлсету

Якщо потрібен швидкий старт, можна створити простий тайлсет з суцільними кольорами:

1. Створіть файл 256x128
2. Намалюйте 4 ромби суцільними кольорами:
   - Зелений ромб (0, 0)
   - Синій ромб (128, 0)
   - Бежевий ромб (0, 64)
   - Коричневий ромб (128, 64)
3. Збережіть як PNG

Це буде працювати, хоч і виглядатиме просто!
